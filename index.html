Rating System
RD
Rating Deviation
RD measures confidence in a player's rating. Low RD = reliable rating; high RD = more uncertainty. New players start at RD 120. Playing a match immediately drops RD — once it falls below 100 you appear on the leaderboard.
⏱ RD
Continuous RD Decay
RD rises continuously while you're inactive, following an asymptotic curve that approaches 120 but never exceeds it. The longer you go without playing, the closer to 100 your RD gets — at which point you fall off the leaderboard.
R↓
Rating-Scaled Decay
Higher-rated players decay more slowly. The decay rate scales as (1500 / rating)^1, so a 1500-rated player decays at 1× while a 2000-rated player decays slower — protecting earned rankings.
Season Settings

Set a season end date. When the deadline passes, you can trigger a season reset — all ratings return to 1500, RD to 120, wins/losses to 0, and a snapshot of final rankings is saved to each player's history.

Restore Previous Season

Accidentally reset a season? Select a past season to restore — this will recreate all matches from that season's archived history, restore each player's final stats, and roll back the season number. Current live matches will be cleared.

GitHub Backup Settings ▼

RD Decay Settings

Rating Snapshot

Saves a snapshot of every player's current rating, RD, wins, and losses. When ratings are recalculated, the snapshot acts as the starting point — all matches before it are ignored, and only matches after it are replayed. Also saved as a standalone backup you can restore.

Repair Ratings

Recalculates every player's rating and RD directly from their last stored match result, then applies any accumulated decay since that match. Use this to fix any players whose values were incorrectly reset.

Deduplicate Matches

Scans the match log for duplicate entries (same two players, same winner, same minute). Keeps exactly one copy of each match and deletes the rest. Run Repair All Ratings afterward to fix any rating drift.

Fix Match Sides

Fixes matches where p1/p2 sides got swapped during a restore — causing LOSSes to show positive rating changes. Ensures p1 always = winner in every match record. Run Repair All Ratings after.

Match Result Simulator

Simulate Glicko-2 outcomes without affecting real data. Load a real player or enter values manually.
